# 利用規約: エディタ用 Azure OpenAI（コーディングエージェント / ログ解析）

社内メンバーがエディタ（Zed / VS Code）から利用する Azure OpenAI エンドポイントの利用条件。

## 利用条件

1. **接続元**: オフィス回線または会社 VPN 経由のみ（IP allowlist 制限）。
   自宅回線・モバイル回線からの直接接続は不可
2. **用途**: 社内の開発業務（コーディング支援）およびログ解析。deployment を用途どおりに使う
   - `agent-main` — コーディングエージェント
   - `log-analysis` — ログ解析
3. **キーの取り扱い**（違反した場合は利用停止）:
   - キーの共有・社外持ち出しの禁止
   - `settings.json` / dotfiles へのキー平文記載の禁止（エディタの keychain 保管機能を使う）
   - リポジトリへのコミット禁止
   - 画面共有・スクリーンショットにキーを写さない
4. **キーは毎週月曜 09:00 JST にローテーションされる**。Teams 通知を受けたら 1 週間以内に更新すること
   （旧キーは次回ローテーションで失効）

## 入力してよいデータ

- 社内コード・設定・ログは入力してよい（Azure OpenAI はプロンプトをモデル学習に使用しない構成）
- ただし**個人情報・認証情報（トークン・パスワード・接続文字列）はマスキングしてから**入力すること
- 顧客から預かったデータは、当該顧客との契約上の制約を確認してから入力すること

## ログ記録について（重要）

本エンドポイントは api-key 認証のため、**呼び出した個人は特定されない**。
記録・監査されるのは以下の範囲であることに同意のうえ利用すること:

- 呼び出し元 IP アドレス
- 使用した deployment（用途）
- トークン量・リクエスト数

多人数運用で問題（コスト急増・不正利用の疑い）が発生した場合、利用者別キー分離
（APIM subscription key 化）への移行を検討する。

## コスト

- 月次予算に対しソフトリミット（50/75/90% で Teams 通知）とハードリミット（100% で**全停止**）が設定されている
- ハードリミット発動中は全員が利用不可になる。大量のトークンを消費する使い方
  （巨大ファイルの全文貼り付け・無人での連続実行）は避けること
